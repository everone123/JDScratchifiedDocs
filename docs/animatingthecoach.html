4 - Animating the Coach

Tip

This page marks the point where we're going to actually start running the project to test things. As such, I recommend going into the "Debug Tools" sprite (near the bottom of the list) and setting both "fast startup?" and "disable webcam in-game?" to 1. Your life will be made MUCH easier.

For our purposes, "animating" the coach will mean making the frames of the GIF play in the ways you want them to. Before we can start doing this, we need to go over some basics.

Custom Blocks in the Coach Sprite​

The sprite containing your GIF (from now on, we'll refer to it as the "coach sprite") contains several custom blocks with varying uses:

This might look overwhelming, but right now we just need to focus on two of them: "do move" and "increase targetbeat". You will build the code for the animation using these two blocks.

Initial Setup​

Locate the coach sprite's main script. This should be near the bottom, and it starts with a "when I recieve load in" block. For your convenience, I have added a code comment that says "main script" directly above it.

Temporarily make the coach sprite visible, then adjust its size and position until you've got it where you want it to be on the screen. Enter those same size and position values into the "set size to" and "go to x: y:" blocks near the top of the main script.

Then, grab a new "set variable" block and use it to change the value of "localCodename" to your song's codename. Remember that it needs to match the file name of the GIF and is case sensitive. You can delete the block after you use it.

Finally, click the green flag, select your song from the menu, and start the routine. Once the loading screen is no longer visible at all, click the stop sign button. We've now initialized the in-game scene, so we can start working with & testing animation blocks.

Remember this!

While the game is in this state, you can do a "test run" at any time by simply clicking on the main script in the coach sprite. Lyrics and motion detection will not work, but most other elements will.

Applying Animation: Trimmed GIF​

If you applied trimming in part 2 to make your GIF take up less space, then this is the part where we'll "put the footage back together". You'll do this in the main script with custom blocks.

You'll need to know what the custom blocks actually do, so here's a more detailed explanation:

"(x)" denotes arguments where you can enter numbers or text.

Custom BlockExplanation
do move (1) (2) (3)Plays frames of the GIF (at 15fps), starting at the frame specified in 2 and ending at the frame specified in 1. Enter "yes" into 3 if you want the block to wait until the target beat is reached (see below) before continuing to the next block.
increase targetbeat (1)The "target beat" variable starts at 0 at the beginning of the song, and increases by the number you enter in 1 whenever you use this block. To keep sync, any running "do move" block will instantly skip ahead if the target beat is ever less than the "beatHalf" variable (which also starts at 0, and increases by 1 every time the pictogram bar flashes).
test move (1) (2)Works like "do move", but completely ignores the target beat (so it will always work when clicked on as an orphaned block). Use this to figure out what frames to use in a "do move" block.
test resetYou can place this block at the start and end of any arrangement of "test move" blocks; after running those blocks, the "!beat test" variable will report the exact number of beats that the animation takes to play out.
Caution

The target beat only supports precision of up to half a beat. For example, "32" and "32.5" are both valid, but "32.25" is not.

All of this is probably confusing. I don't blame you. To help understand it better, let's consider the blocks as "sequences":

Each sequence consists of an "increase targetbeat" block followed by one or more "do move" blocks. The "do move" blocks should be set up to take exactly the number of beats entered into the "increase targetbeat" to play out.

Note that only the last block in a sequence has "yes" entered into the third field. This way, if the sequence accidentally ends too early, it will stall until the target beat is reached.

Add sequences like this to the main script to build the animation. Don't forget that you can click on the main script to test it.

You can insert arbitrary code into the main script if you want to do other stuff to the coach sprite at certain points, but make sure you know what you're doing.

Fixing Music Desync​

If the beats of the song don't match up with the changing of the "!beat" variable (and you're sure you entered the correct tempo earlier), then the music is desynced. Fortunately, there are ways to fix it.

If the song plays too early, adjust the "Song Delays" list value (measured in seconds, decimals supported) to make it line up correctly.

If the song plays too late, add a "wait x seconds" block to the main script after the "start" message is broadcasted, but before the first "do move" block. Increase the value of the first "increase targetbeat" block to account for this delay.
