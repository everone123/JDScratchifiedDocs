2 - Preparing the GIF

To import video footage of a routine into Just Dance Scratchified, you'll need to convert it to a GIF image first. There are some specific requirements that the GIF must meet to be compatible with JDS, so follow this section carefully.

Trimming the Footage​

This step is optional, but is used by all mainline JDS projects and is recommended to save file space, especially if you're porting an official JD routine.

Heads-up

As a rule of thumb, GIFs around 30MB or higher will typically fail to import into any version or mod of Scratch (even offline).

If trimming seems too complicated, do the other steps first, and produce a GIF using the original uncut video; you can come back here if it turns out that you actually need to trim it down.

For our purposes, "trimming" the routine means using a video editor to remove sequences of moves that are exactly the same as moves that were done earlier. The goal is to take out what we can but make sure we can still "piece together" the entire original routine by repeating what we have. As such, you'll also want to keep in any unique transitions between moves.

Depending on how many cuts you make, this will make a later step somewhat more complicated, but the payout of easier importing and less lag is often worth it. When you're done, you can export it as a video (resolution doesn't matter; it'll be changed when we convert it to a GIF).

Applying the Alpha Mask​

If you're using an unmasked routine, skip this section.

Some (but not all) video editing software are capable of both applying masks and exporting as a transparent GIF. In Cosmunn's video descriptions, there are links to three different tutorials for applying an alpha mask; for your convenience, I have provided these same tutorials below.

Before exporting, ensure you've applied the alpha mask properly so that the transparent background looks correct. (Premiere Pro will show the background as black in the video preview, but the background in the actual result will be transparent.)

Exporting​

Your GIF should be created at 15 frames per second. The resolution is up to you, but I typically use 480x270 for the optimal balance of quality and file size.

Note

Premiere Pro does not allow GIFs to export at precisely 15FPS. As a workaround, I instead export as a sequence of PNG images at that framerate and then create a GIF from them using ezgif.com. (For technical reasons, we can't import the PNG frames directly into the project.)

The name of your GIF file should be a short "codename" for your song. Try to keep it under 8 characters (not including the .gif extension).
